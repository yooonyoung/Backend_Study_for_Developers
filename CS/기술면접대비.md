### 마이크로서비스 아키텍처(MSA)란 무엇인가요?

<details>
   <summary> 예비 답안 보기 (👈 Click)</summary>
    <br/>
    MSA는 어플리케이션의 단위를 '목적'으로 나누는 것입니다.<br/>
    모든 기능을 하나의 어플리케이션에서 비즈니스 로직을 구성해 운영하는 Monolithic 아키텍처와 달리, 기능(목적)별로 컴포넌트를 나누고 조합할 수 있도록 구축합니다. MSA에서 각 컴포넌트는 API를 통해 다른 서비스와 통신하는데, 모든 서비스는 각각 독립된 서버로 운영하고 배포하기 때문에 서로 의존성이 없습니다. 따라서 하나의 서비스에 문제가 생겨도 다른 서비스에 영향을 끼치지 않습니다.
</details>

### 트랜잭션이란 무엇인가요?

### 트랜잭션의 특징은?

<details>
   <summary> 예비 답안 보기 (👈 Click)</summary>
    <br/>
    원자성은 트랜잭션이 데이터베이스에 모두 반영되거나, 아니면 전혀 반영되지 않아야 한다는 것이다.<br/>
    일관성은 트랜잭션의 작업 처리 결과가 항상 일관성이 있어야 한다는 것이다.<br/>
    독립성은 어떤 트랜잭션도 다른 트랜잭션의 연산에 끼어들 수 없다는 점이다. <br/>
    지속성은 트랜잭션이 성공적으로 완료되었다면, 결과는 영구적으로 반영되어야 한다는 점이다.
</details>

### GET 방식과 POST 방식의 차이

<details>
   <summary> 예비 답안 보기 (👈 Click)</summary>
    <br/>
    GET 방식은 URL 뒤에 ?를 붙여 데이터를 전달하며, 전송량에 제한이 있고 사용자에게 데이터가 노출됩니다. URL 형식에 맞는 인코딩이 필요합니다. <br/>
    POST 방식은 데이터를 HTTP body에 포함하여 전달하며, 전송량에 제한이 없고 사용자에게 데이터가 보이지 않습니다. GET 방식과 달리 브라우저에서 캐싱(Caching)이 불가능합니다. GET으로 구현할 경우 웹 크롤러가 접근하여 마음대로 실행할 가능성이 있습니다. <br/>
    <br/>따라서 서버의 값이나 상태를 변경하는 경우에는 POST로 구현하고, SQL 쿼리의 SELECT와 같은 단순 조회를 구현할 때에는 GET을 사용하는 것이 좋습니다.
</details>

    <br/><br/>

### 쿠키(Cookie) vs 세션(Session)

<details>
   <summary> 예비 답안 보기 (👈 Click)</summary>
    <br/>
    쿠키와 세션을 사용하는 이유는 HTTP의 비연결성, 비상태성이라는 특징을 보완하기 위함입니다.<br/>
    <br/>
    쿠키는 서버가 사용자의 웹 브라우저, 즉 클라이언트에 저장하는 데이터입니다. 이름, 값, 만료일, 경로 정보로 구성된 text로 저장되며 용량 제한이 있습니다.<br/>
    세션은 일정 시간동안 같은 사용자, 브라우저로부터 들어오는 일련의 요구사항을 하나의 상태로 보고, 그 상태를 일정하게 유지시키는 기술입니다. 세션은 웹 서버에 object 형태로 저장되며, 저장 데이터에 제한이 없습니다. <br/><br/>
    쿠키는 세션보다 빠르지만, 보안은 세션이 쿠키보다 좋습니다. 쿠키는 쿠키 저장 시 설정된 만료 시점에 삭제되고,(만료 시점이 지나지 않았다면 브라우저 종료하더라도 삭제되지 않음) 세션은 브라우저 종료 시 삭제됩니다. 
</details>

### DB에서 인덱스를 사용하는 이유

<details>
   <summary> 예비 답안 보기 (👈 Click)</summary>
    <br/>
    DBMS에서 인덱스는 검색 속도를 높이기 위한 기술입니다. 인덱스는 정렬된 상태를 유지하기 때문에 원하는 값을 탐색하는데는 빠르지만 새로운 값을 추가, 삭제, 수정하는 경우에는 쿼리문 실행 속도가 느려집니다. 즉, DBMS에서 인덱스는 데이터의 저장 성능을 희생하고 그 대신 데이터의 읽기 속도를 높이는 기능이라고 할 수 있습니다.<br/><br/>
    인덱스를 사용하면 좋은 경우는 WHERE 절에서 자주 사용하는 컬럼, 외래키가 자주 사용되는 컬럼, JOIN에 자주 사용되는 컬럼이 있습니다. 반면 데이터 중복도가 높은 컬럼과 DML이 자주 일어나는 컬럼은 인덱스 사용을 피하는 것이 좋습니다.
</details>
